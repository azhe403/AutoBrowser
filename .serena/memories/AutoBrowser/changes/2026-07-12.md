# Changes 2026-07-12

## Theme Fix — Startup Order (Gallery Pattern)
- Moved `ApplyTheme()` from `App.xaml.cs` (before `ShowMainWindow`) to `MainWindow_Loaded` event in `App.Window.cs`
- `ApplicationThemeManager.Apply()` requires `MainWindow` to exist to update window background/resources
- Follows WPF UI Gallery pattern: `App.xaml` sets initial theme via `<ui:ThemesDictionary Theme="Light"/>`, theme applied after window creation

## Dark Theme Typography Fix — All Pages
- Added explicit `Foreground="{DynamicResource TextFillColorPrimaryBrush}"` to ALL `ui:TextBlock` with `FontTypography` across all pages
- `FontTypography` alone does NOT inherit theme foreground — WPF defaults to black
- Fixed in: HomePage.xaml, SettingsPage.xaml, AboutPage.xaml, RulesListView.xaml
- Secondary/description text uses `TextFillColorSecondaryBrush`
- Description text in About page card also needed explicit `TextFillColorSecondaryBrush`

## Namespace Cleanup — Remove Fully Qualified Names (Comprehensive)
- Replaced ALL long fully qualified namespace references with `using` directives across entire codebase
- Created `GlobalUsings.cs` with aliases for ambiguous types (Application, Brush, Color, FontFamily, UserControl, MessageBoxButton, MessageBoxImage)
- **Rule**: No local `using` aliases — all aliases go in `GlobalUsings.cs` only
- Files cleaned: App.Tray.cs, App.xaml.cs, App.Window.cs, App.Registration.cs, HomeViewModel.cs, MainViewModel.cs, AboutViewModel.cs, SettingsViewModel.cs, RuleEditorView.xaml.cs, RuleTesterView.xaml.cs, RulesListView.xaml.cs, ToolbarView.xaml.cs, StatusControl.xaml.cs, SingleInstanceService.cs, UpdateService.cs
- Replaced long fully qualified namespace references with `using` directives
- Added `using System.Drawing;`, `using System.Windows.Forms;` to `App.Tray.cs`
- Added `using System.Threading;`, `using System.Windows;`, `using System.Windows.Forms;` to `App.xaml.cs`
- Added `using Application = System.Windows.Application;` alias to resolve ambiguity between WPF and WinForms Application types
- Added `using System.Drawing;`, `using System.Windows.Forms;` to `HomeViewModel.cs`
- Changed `System.Windows.Application` base class to use alias `Application`
