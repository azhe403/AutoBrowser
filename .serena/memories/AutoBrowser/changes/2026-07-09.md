## Infrastructure & Configuration
- **Onboarding completed** — Created standard local memories (`core`, `tech_stack`, `suggested_commands`, `conventions`, `task_completion`).

## Refactoring MainWindow to MVVM & DI
- **WPF UserControls Slicing** — Segmented the monolithic `MainWindow.xaml` into modular, reusable UserControls under `Views/` (`ToolbarView`, `RulesListView`, `FooterView`, `StatusControl`).
- **DI Registration** — Registered all services, `MainWindow`, and `MainViewModel` in a global `ServiceProvider` in `App.xaml.cs`.
- **Zero Code-Behind** — Stripped down `MainWindow.xaml.cs` to a boilerplate constructor that resolves the `MainViewModel` from the global `ServiceProvider`.
- **WPF App Lifecycle Consolidation** — Consolidated window state saving/restoring, tray icon management, URL command-line parameter routing, and app path migration check logic into `App.xaml.cs`.
- **RoutingRule Property Tracking** — Refactored `RoutingRule.cs` to inherit from `ObservableObject` and utilize properties generated via MVVM source generators. Configured `MainViewModel.cs` to dynamically auto-save rules on collection or property change events.
- **Double-Click Edit Fix** — Moved double-click `MouseBinding` to `<Border.InputBindings>` inside the `DataTemplate` in `RulesListView.xaml` to resolve event bubbling issues.
- **Command CanExecute Notification Fix** — Added `[NotifyCanExecuteChangedFor(...)]` to `_selectedRule` property in `MainViewModel.cs` to trigger `NotifyCanExecuteChanged()` on `EditRuleCommand`, `DeleteRuleCommand`, `MoveUpCommand`, and `MoveDownCommand`. This enables and disables the rule management toolbar buttons dynamically based on item selection.
- **Pretty Dialog & Sequential Dialog Flow** — Redesigned the path mismatch warning dialog to use aligned WPF controls and structured theme-conforming layouts. Changed the dialog flow to execute sequentially (awaiting the re-registration dialog before starting the update checker), preventing overlapping dialog boxes on startup.
