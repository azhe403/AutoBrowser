# 2026-07-09 Changes

## Infrastructure & Configuration
- **Onboarding completed** — Created standard local memories (`core`, `tech_stack`, `suggested_commands`, `conventions`, `task_completion`).
- **Ignore `.idea/` completely** — Updated root `.gitignore` to ignore the JetBrains `.idea/` folder and untracked existing cached files to eliminate `git status` noise.
- **Dependency Injection** — Added `Microsoft.Extensions.DependencyInjection` NuGet package reference to `AutoBrowser.csproj`.
- **Pre-Build Process Termination** — Added `KillBeforeBuild` MSBuild target inside `AutoBrowser.csproj` to execute `taskkill /F /IM AutoBrowser.exe` via `cmd /c` (safely returning exit code 0) prior to building, preventing file access/write locks during development cycles.

## Refactoring MainWindow to MVVM & DI
- **WPF UserControls Slicing** — Segmented the monolithic `MainWindow.xaml` into modular, reusable UserControls under `Views/` (`ToolbarView`, `RulesListView`, `FooterView`, `StatusControl`).
- **DI Registration** — Registered all services, `MainWindow`, and `MainViewModel` in a global `ServiceProvider` in `App.xaml.cs`.
- **Zero Code-Behind** — Stripped down `MainWindow.xaml.cs` to a boilerplate constructor that resolves the `MainViewModel` from the global `ServiceProvider`.
- **WPF App Lifecycle Consolidation** — Consolidated window state saving/restoring, tray icon management, URL command-line parameter routing, and app path migration check logic into `App.xaml.cs`.
- **RoutingRule Property Tracking** — Refactored `RoutingRule.cs` to inherit from `ObservableObject` and utilize properties generated via MVVM source generators. Configured `MainViewModel.cs` to dynamically auto-save rules on collection or property change events.
- **Double-Click Edit Fix** — Moved double-click `MouseBinding` to `<Border.InputBindings>` inside the `DataTemplate` in `RulesListView.xaml` to resolve event bubbling issues.
- **Command CanExecute Notification Fix** — Added `[NotifyCanExecuteChangedFor(...)]` to `_selectedRule` property in `MainViewModel.cs` to trigger `NotifyCanExecuteChanged()` on `EditRuleCommand`, `DeleteRuleCommand`, `MoveUpCommand`, and `MoveDownCommand`. This enables and disables the rule management toolbar buttons dynamically based on item selection.
- **Pretty Dialog & Sequential Dialog Flow** — Redesigned the path mismatch warning dialog to use aligned WPF controls and structured theme-conforming layouts. Changed the dialog flow to execute sequentially (awaiting the re-registration dialog before starting the update checker), preventing overlapping dialog boxes on startup.

## Code Cleanups & File Splitting
- **Class Decomposition** — Split the large `App.xaml.cs` (600+ lines) into multiple partial C# source files to keep the main shell clean and modular:
  - `App.xaml.cs`: Holds app initialization, DI config, and WPF startup/exit lifecycle handlers.
  - `App.Tray.cs`: Hosts system tray icon creation, double-click behavior, and notification balloon tip logic.
  - `App.Routing.cs`: Hosts single-instance named pipe server, command-line argument checking, and URL routing interceptor dispatch.
  - `App.Registration.cs`: Hosts the protocol/default browser registration validation and path change prompt.
  - `App.Window.cs`: Hosts the `MainWindow` Loaded, StateChanged, Closing lifecycle handlers and settings-based window bounds persistence.
- **Dev-Mode CLI Detection** — Configured `CheckAndPromptReRegister` to check if the running process is actually `AutoBrowser.exe` (ignoring CLI wrappers like `dotnet.exe`), avoiding the "Path Changed" popups during debug sessions (`dotnet run`).
- **Startup Geometry Adjustments** — Synchronized default window dimensions (`1075` x `725`) directly into `AppSettings.cs` as the first-launch fallback values.

## Title Bar & Navigation Restoration
- **Title Bar Drag & Visuals Fix** — Restored `ExtendsContentIntoTitleBar="True"` and the `<ui:TitleBar>` control in `MainWindow.xaml`. To prevent focus and dragging issues, wrapped both `ui:TitleBar` and `ui:NavigationView` in a root `Grid` container layout (placing `ui:TitleBar` in Row 0 and `ui:NavigationView` in Row 1), which allows `ui:TitleBar` to correctly locate and register the WindowChrome hooks.

## About Page Credits & NuGet Reference List
- **Third-Party Attribution** — Added a structured, modern credits card section to `AboutPage.xaml` listing all primary NuGet packages used by the application (`WPF-UI`, `CommunityToolkit.Mvvm`, `Serilog`, `Microsoft.Extensions.DependencyInjection`) along with description and project links.
