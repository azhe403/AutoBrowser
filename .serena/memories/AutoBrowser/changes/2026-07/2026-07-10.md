# Changes 2026-07-10 (Update 1)

- Added `Serilog.Enrichers.Thread` NuGet package to `AutoBrowser.csproj`.
- Updated logging configuration in `App.xaml.cs` to include `.Enrich.WithThreadId()` and output `[{ThreadId}]` in both console and file templates.
- Fixed StatusControl binding in MainWindow.xaml by handling the RootNavigation.Navigated event in MainWindow.xaml.cs to dynamic apply page DataContext to StatusView.
- Updated SettingsViewModel.cs to use safe cast (`as App`) when accessing `System.Windows.Application.Current` to fix unit tests.
- Removed dummy `App` initialization in unit tests (MainViewModelTests.cs) as they were causing AppDomain exception.
- Fixed exit hangs by invoking `Current.Shutdown()` inside both MainWindow's closing handler (when not closing to tray) and the tray context menu exit handler to fully terminate the application process.
- Fixed `App.Registration.cs` to check `_defaultBrowserService.GetRegisteredPath()` directly instead of relying on `IsDefaultBrowser()` which fails early when paths mismatch.
- Updated path resolution for `oldPath` to check `_protocolService.GetRegisteredPath()` and `_defaultBrowserService.GetRegisteredPath()` directly instead of checking `IsProtocolRegistered()` / `IsDefaultBrowser()` (which return false on path mismatch).
- Added bottom margin/padding to `HomePage.xaml`, `SettingsPage.xaml`, and `AboutPage.xaml` to prevent content overlapping with the overlaid `StatusControl`.
- Moved the `StatusControl` element completely outside of the `ui:NavigationView` in `MainWindow.xaml`. Placed it directly as a sibling under the root `Grid` to prevent it from overlaying on top of page content/scrollbars inside the Navigation frame. Added appropriate margins to clear the LeftFluent side navigation pane.
- Removed bottom Margin `70px` from the main `Grid` layout panels in `HomePage.xaml`, `SettingsPage.xaml` and `AboutPage.xaml` which was clipping `ScrollViewer` track heights. Applied bottom margins directly on the scrolling child panels (`StackPanel`/`Grid` inside `ScrollViewer`) to preserve full-size scroll track dragging while keeping visual spacing for the floating `StatusControl`.
- Added bottom padding `16` to `ui:ListView` inside `RulesListView.xaml` to prevent rules scrolling content from clipping.
- Adjusted status bar clearance margin to `72` on left to perfectly clear the NavigationView LeftFluent sidebar.
- Replaced the custom status border with a native `ui:InfoBar` in `StatusControl.xaml` to display status updates inside a standard, styled informational notification bar.
- Moved `StatusControl` from `NavigationView.ContentOverlay` to become a direct sibling of `NavigationView` in the root `Grid` of `MainWindow.xaml`. This ensures visual separation from the navigation host frame.
- Enabled mouse transparency on `StatusControl` by setting `IsHitTestVisible="False"` on the root `UserControl` but keeping it `True` for the inner `ui:InfoBar`, allowing mouse wheel and click events to pass through to the page content beneath the transparent areas.
- Added standard `ScrollViewer` wrapper around the content areas of `SettingsPage.xaml` and `AboutPage.xaml` inside their respective Grid layouts.
- Bound the startup initialization checks (silent updates, path re-registration, CLI URL dispatching) to the `MainWindow.ContentRendered` event instead of a timer, ensuring the main window layout is 100% painted and visible before any blocking dialog runs.
- Fixed non-movable title bar bug after `MessageBox` closes by explicitly focusing the `MainWindow` upon dialog completion.
- Fixed `DefaultBrowserService.IsDefaultBrowser` to compare the actual registered `shell/open/command` executable path against the current process path instead of just checking if the generic application name key exists, preventing false positives when the app is moved.
