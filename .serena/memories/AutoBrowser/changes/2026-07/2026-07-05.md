# Changes 2026-07-05

## Serena MCP integration
- New `opencode.json` at project root — configures Serena MCP server for semantic code analysis
- Command: `serena start-mcp-server --project-from-cwd` (auto-detects project)
- Requires `serena-agent` installed via `uv tool install -p 3.13 serena-agent`
- Initialized with LSP backend (`serena init`)

## Bug fixes
- `src/AutoBrowser/App.xaml.cs` — `ApplyTheme()` now loads existing settings before saving, preventing theme mode from overwriting other settings
- **AutoUpdater argument parsing fix** — `src/AutoUpdater/Program.cs` was expecting positional arguments (`args[0]`, `args[1]`, `args[2]`) but `UpdateService` was sending named arguments (`--pid`, `--source`, `--target`, `--exe`, `--workspace`). Added `ParseArguments()` method to properly handle named arguments. This was causing the auto-update to silently fail because the AutoUpdater couldn't parse the PID, source, or exe path correctly.

## AutoUpdater enhancements
- **Added Serilog logging** — `src/AutoUpdater/Program.cs` now uses Serilog with file sink
  - Removed `PublishAot=true` from csproj since Serilog requires reflection
  - Added `Serilog` and `Serilog.Sinks.File` NuGet packages
  - Logs to `Logs/updater-*.log` in same directory as the app exe
  - Rolling logs with 7-day retention
  - Removed unused `FileLogger.cs` (replaced by Serilog)
- **Backup directory moved to app dir** — `UpdateBackup/` folder next to the exe instead of `%TEMP%`
- **Single-file publishing** — AutoUpdater now publishes as a single .exe file
  - Added `PublishSingleFile=true` and `SelfContained=false` to csproj
  - Updated main AutoBrowser.csproj to use `dotnet publish` instead of `dotnet build` for updater
  - Result: single `AutoUpdater.exe` (~374KB) with all dependencies bundled
